vue-router push 不更新缓存页面.

Posted by     "" on Wednesday, March 15, 2023

在开发中遇到一个问题,第一步,搜索页面点击跳转到某一个详情后,希望可以返回时缓存搜索页面.直接回到搜索结果还在的页面,而不是重新加载页面.这个时候就需要使用 vue-router 的缓存功能了. http://aqingya.cn/articl/5d2ddfae.html

//打开router.js
//在需要缓存的路由页面中,添加meta属性
export default new Router({
  routes: [
    {
      path: '/hello',
      name: 'HelloWorld',
      component: HelloWorld,
      meta: {
        keepAlive: true // 需要缓存
      }
    },
    {
      path: '/hello2',
      name: 'HelloWorld2',
      component: HelloWorld2,
      meta: {
        keepAlive: false // 不需要缓存
      }
    }
  ],

//修改App.vue
<template>
  <div id="app">
    <!-- <router-view /> -->
    <keep-alive>
      <router-view v-if="$route.meta.keepAlive"></router-view>
    </keep-alive>
    <router-view v-if="!$route.meta.keepAlive" />
  </div>
</template>

这样就可以实现页面的缓存.但这会出现一个新的问题(不太确定二者是不是有因果关系).当我在搜索页面,点击一个详情查看后,再回到搜索详情页,点击下一个页面,发现这个详情页是缓存的之前的详情页,没有更新成新点击的这个详情页.原因是因为用了 this.$router.push 解决方法: (1).给keepalive 设置:exclude=“exclude"和:key="$route.fullPath”(实测发现只需要设置:exclude=“exclude"就够了,不需要设置:key https://blog.csdn.net/leveretz/article/details/127834566 https://blog.csdn.net/weixin_47800450/article/details/112167359?utm_medium=distribute.pc_relevant.none-task-blog-2~default~baidujs_baidulandingword~default-0-112167359-blog-128286453.235^v38^pc_relevant_sort_base2&spm=1001.2101.3001.4242.1&utm_relevant_index=3

//App.vue
<keep-alive :exclude="exclude">
        <router-view :key="$route.fullPath"  v-if="isRouterAlive"  style="margin: 0 auto;" />
 </keep-alive>
//App.vue下面的data中定义exclude,写不想被缓存的路由 data() { return { exclude: ['/search', '/searchDetail'], } },

(2)添加 date:new Date().getTime()

this.$router.push({
  path: '/homePage/searchResult',
  query: {
    keywords: this.input,
    type: this.type,
    date: new Date().getTime(),
  },
})
