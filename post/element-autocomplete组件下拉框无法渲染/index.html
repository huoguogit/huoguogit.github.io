element Autocomplete组件下拉框无法渲染

Posted by     "huoguo32" on Wednesday, March 8, 2023

element的Autocomplete组件下拉框无法渲染,但是输入框可以正常输入,并有数据,查阅资料显示

  const querySearchAsync = (queryStr, cb) => {
   
      placeSearch.search(queryStr, (status, result) => {
        // console.log(status, result)
        if (status === 'complete') {
          searchResult.value = result.poiList.pois
          cb(searchResult.value)
        }
      })
    }

如果el-autocomplete接受的数组对象里面没有value属性，就需要用value-key（输入建议对象中用于显示的键名）可以指定对应的key，如果不用value-key,那就需要处理数组，给数组加上一个value的key，要不然输入建议显示不出来，如下图

  <el-autocomplete
      v-model="inputValue"
      :fetch-suggestions="querySearchAsync"
      placeholder="请输入关键词"
      //因为请求回来的数据是数组对象,所以取每个对象中的name值,这样下拉就会渲染name
      value-key="name"
      :trigger-on-focus="false"
      popper-class="autocompletePopper"
      :popper-append-to-body="false"
      @select="handleSelect"
      class="my-autocomplete"
    ></el-autocomplete>

这个 value-key 属性的默认值是 value，你的键名不一定是叫value，改成你自己对应的就可以了
