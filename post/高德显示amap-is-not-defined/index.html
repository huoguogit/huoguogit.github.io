高德显示AMap is not defined

Posted by     "huoguo32" on Wednesday, March 15, 2023

高德在vue中常见的的组件加载问题,AMap is not defined,直接这么写会报错,

   const districtSearch = new AMap.DistrictSearch({
        subdistrict: 1,
        extensions: 'all',
        level: 'city',
        showbiz: false,
        city
      })

      districtSearch.search(city, (status, result) => {
        console.log(city, status, result)
        if (status === 'complete') {
          districtList.value = result.districtList[0].districtList
        }
      })
    })

解决方法: 1(重点):先检查之前的AMapLoader.load的 plugins有没有加载所需的组件,之前就是因为忘了加载这个’AMap.DistrictSearch’,所以卡了好久

 AMapLoader.load({
    key: 'ea42979a6cc0fa36a0d48a9004f59504', // 申请好的Web端开发者Key，首次调用 load 时必填
    version: '2.0', // 指定要加载的 JSAPI 的版本，缺省时默认为 1.4.15
    plugins: [
      'AMap.Scale',
      'AMap.ToolBar',
      'AMap.AutoComplete',
      'AMap.PlaceSearch',
      'AMap.ControlBar',
      'AMap.MouseTool',
      'AMap.MapType',
      'AMap.HawkEye',
      'AMap.DistrictSearch'
    ],
    Loca: {
      version: '2.0.0'
    }
  })

2.在子组件使用时,使用AMapLoader异步加载AMap库

<!-- 必须先引入AMapLoader -->
import AMapLoader from '@amap/amap-jsapi-loader'
 const searchDistrict = (city) => {
  AMapLoader.load({})
    .then(() => {
      const districtSearch = new AMap.DistrictSearch({
        subdistrict: 1,
        extensions: 'all',
        level: 'city',
        showbiz: false,
        city
      })

      districtSearch.search(city, (status, result) => {
        console.log(city, status, result)
        if (status === 'complete') {
          districtList.value = result.districtList[0].districtList
        }
      })
    })
    .catch((err) => {
      console.error(err)
    })
}
