高德渲染图标概率性报错(不显示)

Posted by     "" on Tuesday, March 14, 2023

新汴河大屏项目.发现根据请求数据加载网络点位时,有的时候能顺利加载点位,有的时候无法加载

打印加载失败的报错信息,发现都是通知AMap is not defined.

chatGPT解决方案:

根据你提供的信息，“AMap is not defined” 这个错误通常意味着在加载图标之前，AMap JavaScript API 尚未完全加载完成，导致相应的 JavaScript 对象没有被正确地定义。

为了解决这个问题，你可以尝试以下几个步骤：

  1. 确保在加载图标之前，AMap JavaScript API 已经被完全加载。你可以在控制台中输入 AMap 命令，检查是否已经成功加载。

  2. 如果确信 AMap API 已经加载，你可以考虑使用回调函数，确保加载完成后再加载图标。例如：

  3. 如果以上两个步骤均无效，你可以尝试延迟加载图标，等待 AMap JavaScript API 加载完成后再加载。例如：

    (最终采用方法三解决,如果AMap的值不是undifined,再执行请求数据渲染点位的函数)

// 等待 AMap JavaScript API 加载完成后，再加载图标
function loadMarker() {
  if (typeof AMap !== 'undefined') {
    // 加载图标
    // ...
  } else {
    setTimeout(loadMarker, 100);
  }
}

// 开始加载图标
loadMarker();
